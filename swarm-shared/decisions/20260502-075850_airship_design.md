# airship / design

### Highest-Value Incremental Improvement
The highest-value incremental improvement that can ship in <2h is to implement a fix for the HF API rate limit 429 error. This error occurs when the number of requests to the Hugging Face API exceeds 1000 requests per 5 minutes. 

### Implementation Plan
To fix this issue, we will:
1. Modify the `list_repo_files` function to use pagination and avoid recursive calls on big repositories.
2. Use `list_repo_tree(path, recursive=False)` per folder instead of `list_repo_files`.
3. Implement a retry mechanism with a 360-second wait after encountering a 429 error.

### Code Snippets
```bash
# Modify list_repo_files function to use pagination
def list_repo_files(repo_id, path, recursive=False):
    files = []
    for result in hf_api.list_repo_tree(repo_id, path, recursive=False):
        files.extend(result.files)
    return files

# Implement retry mechanism with 360-second wait
def retry_hf_api_call(func, *args, **kwargs):
    retry_count = 0
    while retry_count < 3:
        try:
            return func(*args, **kwargs)
        except HF_API_RateLimitError:
            retry_count += 1
            time.sleep(360)
    raise HF_API_RateLimitError("Rate limit exceeded")
```
### Example Use Case
```python
# Get list of files in a repository
files = list_repo_files("username/repo", "/path/to/folder")

# Use retry mechanism to call HF API
files = retry_hf_api_call(hf_api.list_repo_files, "username/repo", "/path/to/folder")
```
This improvement will help prevent the HF API rate limit 429 error and ensure that the application can continue to function without interruptions. The estimated time to implement this fix is approximately 1 hour, which is well within the 2-hour time frame.
