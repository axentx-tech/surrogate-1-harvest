# surrogate-1 / design

### Diagnosis
* The project lacks a robust implementation for handling Hugging Face API rate limits, which can block dataset training.
* The existing implementation may not be reusing existing Lightning Studio instances efficiently, leading to wasted resources.
* The project does not have a clear strategy for downloading dataset files from the Hugging Face CDN, which can bypass API rate limits.
* The current implementation may be running heavy compute tasks on the Mac, which can lead to performance issues and inefficiencies.
* The project does not have a mechanism to restart training processes if they are stopped due to idle timeouts.

### Proposed change
The proposed change is to implement a robust Hugging Face API rate limit handling mechanism and reuse existing Lightning Studio instances efficiently. This can be achieved by modifying the `train.py` script to use the Hugging Face CDN to download dataset files and reuse existing Lightning Studio instances.

### Implementation
To implement this change, we can follow these steps:

1. Modify the `train.py` script to use the Hugging Face CDN to download dataset files. This can be done by replacing the `load_dataset` function with a custom function that downloads the dataset files from the CDN.
```python
import requests

def download_dataset_files(repo_id, path):
    url = f"https://huggingface.co/datasets/{repo_id}/resolve/main/{path}"
    response = requests.get(url)
    return response.content

# Usage
dataset_files = download_dataset_files("repo_id", "path/to/dataset")
```
2. Reuse existing Lightning Studio instances by listing the running studios and reusing them if they exist.
```python
import lightning

# List running studios
studios = lightning.Teamspace.studios()

# Reuse existing studio if it exists
for studio in studios:
    if studio.name == "studio_name" and studio.status == "Running":
        studio_to_use = studio
        break
else:
    # Create a new studio if none exist
    studio_to_use = lightning.Studio(create_ok=True)
```
3. Modify the `train.py` script to restart training processes if they are stopped due to idle timeouts.
```python
import lightning

# Check if the studio is stopped
if studio_to_use.status == "Stopped":
    # Restart the studio
    studio_to_use.start(machine=lightning.Machine.L40S)
```
