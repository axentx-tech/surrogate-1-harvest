# Data Schema Validation Skill

## Overview
This skill defines a standard process to ensure data quality for JSONL training datasets by:
1. Defining a canonical schema (`data/schema/schema.yaml`).
2. Implementing a validation script (`scripts/validate_schema.py`).
3. Integrating validation into CI pipelines.
4. Tracking a compliance metric (`data_schema_compliance`).

## Steps

1. **Create Schema File**
   ```yaml
   # data/schema/schema.yaml
   version: 1.0
   description: Training data schema
   fields:
     - name: instruction
       type: string
       required: true
     - name: response
       type: string
       required: true
     - name: source
       type: string
       required: false
     - name: repo
       type: string
       required: false
     - name: topic
       type: string
       required: false
   ```

2. **Add Validation Script** (`scripts/validate_schema.py`)
   ```python
   import yaml, json, sys, pathlib
   from jsonschema import validate, Draft7Validator

   SCHEMA_PATH = pathlib.Path(__file__).parents[2] / "data" / "schema" / "schema.yaml"
   DATA_GLOB = "data/raw/**/*.jsonl"

   def load_schema():
       with open(SCHEMA_PATH) as f:
           raw = yaml.safe_load(f)
       # Convert to JSON Schema format
       json_schema = {
           "$schema": "http://json-schema.org/draft-07/schema#",
           "type": "object",
           "properties": {field["name"]: {"type": field["type"]} for field in raw["fields"]},
           "required": [field["name"] for field in raw["fields"] if field.get("required")],
       }
       return json_schema

   def validate_file(path, schema):
       ok = 0
       total = 0
       with open(path) as f:
           for line in f:
               total += 1
               try:
                   record = json.loads(line)
                   validate(instance=record, schema=schema)
                   ok += 1
               except Exception as e:
                   pass
       return ok, total

   if __name__ == "__main__":
       schema = load_schema()
       total_ok, total = 0, 0
       for p in pathlib.Path('.').glob(DATA_GLOB):
           ok, t = validate_file(p, schema)
           total_ok += ok
           total += t
       compliance = (total_ok / total) * 100 if total else 0
       print(f"data_schema_compliance={compliance:.2f}%")
   ```

3. **CI Integration** (example GitHub Actions snippet)
   ```yaml
   # .github/workflows/ci.yml
   name: CI
   on: [push, pull_request]
   jobs:
     validate-data:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v3
         - name: Set up Python
           uses: actions/setup-python@v4
           with:
             python-version: '3.11'
         - name: Install dependencies
           run: |
             pip install pyyaml jsonschema
         - name: Run schema validation
           run: python scripts/validate_schema.py
         - name: Fail on low compliance
           env:
             THRESHOLD: 90
           run: |
             compliance=$(python scripts/validate_schema.py | grep -oP "[0-9.]+" )
             if (( $(echo "$compliance < $THRESHOLD" | bc -l) )); then
               echo "Compliance $compliance% is below $THRESHOLD%" && exit 1
             fi
   ```

4. **Metrics Dashboard**
   Emit the `data_schema_compliance` metric from the validation script and ingest it into any existing monitoring system (e.g., Prometheus, Grafana, or a custom dashboard).

## Pitfalls & Tips
- Keep `schema.yaml` version‑controlled; bump `version` when fields change.
- Run the validation script locally before committing large data batches.
- Adjust the CI compliance threshold based on project maturity.
