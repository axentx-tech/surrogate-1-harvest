## Key Conventions
- **Buildspec location**: `template/{stack-name}-buildspec.yaml` (NOT root)
- **Params format**: `[{"ParameterKey":"","ParameterValue":""}]`
- **Versioning**: Build number via SSM `VERSION_BUILDNUMBER_PARAM` → `major.minor.BUILD_NUMBER`
- **Cross-region sync**: FATAL on failure, NEVER warn-only
- **Secrets**: NEVER bake into Docker. Inject at runtime via ECS Task Definition + Secrets Manager
- **Shared cluster**: go-api intentionally uses nodejs-api ECS cluster (cost saving)
- **DevSecOps pipeline**: gitleaks(warn) → semgrep(warn,resilient) → cfn-lint(warn) → npm audit(block critical) → lint → test → build → checksum → build-info.json → sync → deploy

## Known Gotchas (IMPORTANT — saves hours of debugging)

### 1. semgrep on Amazon Linux 2 CodeBuild
`pip3 install semgrep` installs Python wrapper but `semgrep-core` binary is often missing.
**Fix**: Always check `semgrep --version` works before running scan. Skip with warning if not.
```bash
if command -v semgrep &>/dev/null && semgrep --version &>/dev/null; then
  semgrep --config auto --error --severity ERROR --timeout 120 . || { echo "ERROR: ..."; exit 1; }
else
  echo "WARN: semgrep not available — skipping SAST"
fi
```

### 2. Lambda zip silent failure
`npm ci` in `dist/` needs BOTH `package.json` AND `package-lock.json`. If package-lock.json is missing, npm ci fails, breaks the && chain, and zip is never created — BUT build reports SUCCESS.
**Fix**: Always copy both files AND add guard check:
```bash
cp package.json package-lock.json dist/
# ... after zip command:
if [ ! -f "file.zip" ]; then echo "ERROR: zip not created" && exit 1; fi
```

### 3. IAM out-of-band changes
go-api and nodejs-api CodeBuild roles have inline policy `S3CrossRegionTH` added manually (not in CF template). If CI stacks are recreated, this policy will be lost.
**TODO**: Add to CI CloudFormation templates.

### 4. npm audit false positives
Use `--omit=dev` to exclude dev dependency vulnerabilities. Use `--audit-level=critical` to only block on critical.
car-backend had critical vulns in firebase-admin dependency chain — fixed with `npm audit fix` (non-breaking).

### 5. Docker secrets (NEVER DO THIS)
```bash
# BAD — secrets baked into image layer:
docker build --build-arg DB_HOST=xxx --build-arg DB_PASS=xxx .

# GOOD — runtime injection:
# Docker image has no secrets. ECS Task Definition injects via Secrets Manager.
```
