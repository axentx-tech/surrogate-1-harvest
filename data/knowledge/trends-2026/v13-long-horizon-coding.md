# V13 Long-Horizon Coding — Idea → MVP → V1 → V2 → ... → V10000

> Owner goal: model that **ships products autonomously** without human in the loop.
> Currently ships zero features. V8 = code patches. V11 = code + IaC + memory.
> **V13 = full project lifecycle** — from natural-language idea to deployed app, iterated forever.
>
> Gap to close: SWE-bench Verified is single-issue. Real shipping = multi-file, multi-day,
> multi-iteration, with tests + docs + deploy + postmortem feedback.

## Problem Frame — Why "Ship a Project" ≠ "Solve a PR"

| Capability | SWE-bench (V8/V11) | Real shipping (V13 target) |
|---|---|---|
| Context length | 8-32K tokens | 128K-1M tokens |
| Tool calls per task | 5-20 | 100-500+ |
| Files touched | 1-5 | 10-100+ |
| Wall-clock horizon | minutes | hours-to-days |
| Reward signal | unit tests pass | tests + lint + integration + user accept |
| Iteration count | 1 PR | many versions, postmortems, redesign |
| Recovery | retry once | self-debug, replan, escalate |
| Success bar | patch applied | deployed, used, didn't break in prod |

V11's blend (SWE-smith 8K + R2E-Gym 6K) trains **patch-level** behavior. V13 needs
**project-level** behavior — long trajectories where the model lays out a spec,
implements, tests, deploys, gets feedback, and **iterates** through multiple versions.

Frontier evidence (2026-Q1): GPT-5.4 hits 72.8% on SWE-bench Verified but only **25%**
on SWE-EVO (multi-file evolution) and **23%** on SWE-Bench Pro. The cliff between
"fix one bug" and "evolve a codebase" is the V13 cliff.

## Training Techniques for Very-Long-Horizon Agents

Beyond data, V13 needs scaffolding:

### A. Long-context training infrastructure
- **Sequence packing** to 128K (CoderForge recipe) — already in V11; verify utilization
  hits >85% with new datasets.
- **Context parallelism** to fit 131K-token sequences across GPUs (CWM mid-training pattern,
  Meta FAIR). Skip on Kaggle T4×2 (insufficient interconnect); use on Lightning H200.
- **YaRN / RoPE scaling** beyond Qwen2.5-Coder-7B's native 32K. Enable
  `rope_scaling: yarn factor=4.0` for 128K eval; train at 32K, eval at 128K.

### B. Multi-turn RL on real Git repos (V13.5+)
- **SWE-RL recipe** (Meta FAIR): pull-request reward = `difflib.SequenceMatcher` similarity
  between agent patch and merged-PR golden patch. Continuous 0-1 reward, partial credit.
- **DeepSWE recipe** (Together AI / Agentica): R2E-Gym subset + rLLM framework + GRPO.
  Got Qwen3-32B → 42.2% Pass@1 / 59% test-time-scaled on SWE-bench Verified.
- **Filter rule**: exclude problems from same repos as your eval benchmarks (sympy etc.).
  Contamination kills the signal.

### C. Process reward models / critics
- **AgentPRM**: train a step-level reward model on R2E-Gym verifier trajectories. Critic
  produces dense reward for every action, not just terminal. Sample-efficiency multiplier.
- **Court-style 3-role eval** (Grader / Critic / Defender): for "is this a good MVP?",
  three LMs argue, score is consensus. Owner's V13 question #6 answered.
- **CourtEval pattern** — applicable as inference-time deliberation, not training-time.

### D. Self-bootstrapping pipeline (V13.x)
- **Trajectory database**: every inference run by Surrogate-1 logs to ADP format.
  Success label from: did unit tests pass? Did owner accept? Did deploy succeed?
- **Re-train cadence**: weekly on accumulated successes, weight successful trajectories
  3.0×, retain failure trajectories for recovery training at weight 0.5×.
- **Reference**: AReaL-SEA hierarchical self-evolving pattern. Traj-Bootstrap quality filter.

## Trace Formats — Compatibility With Existing Pipeline

V11 `extract_pair()` handles: ShareGPT, instruction/output, prompt/response, messages-list.
V13 datasets fall into 3 trace shapes — all need handlers:

1. **Single-turn** (OpenCodeReasoning-2, OrcaAgentInstruct): `{prompt, response}` —
   already works with V11's `extract_pair()`.
2. **Multi-turn ReAct** (CoderForge, SWE-rebench OpenHands, SWE-Gym, R2E-Gym, SWE-Dev):
   needs flatten-to-pairs OR train as native multi-turn. Recommendation: flatten in V13.0
   for stability; native multi-turn in V13.5 once long-context training proven.
3. **Action/observation** (computer-use AgentNet/OpenCUA, AppWorld): screenshot+action
   sequences. Skip in V13.0 unless multimodal; defer to V14.

**Adopt Agent Data Protocol (ADP)** as canonical internal format. ADP unified 13 datasets
in their paper → +20% avg gain. Convert all multi-turn datasets through ADP standardizer
before flattening.

## Risk + Mitigation

| Risk | Mitigation |
|---|---|
| 128K-context OOM on Kaggle T4×2 | Train at 32K, eval at 128K with YaRN. Use Lightning H200 for >32K runs. |
| License conflict (private use vs research-only) | SWE-bench Pro is eval-only — never `merge_external` it. Annotate license in env knob name. |
| Trajectory format chaos | Convert to ADP first. Single normalizer in extract_pair. |
| Catastrophic forgetting on V11 strengths | Keep V11 take levels for IaC + Multi-IaC + ITBench. Add V13 on top, not replace. |
| Overfit to SWE-bench style | Add Multi-SWE-RL (multi-lang), Web-Bench-style data when available. |
| Bootstrap loop poisoning (own bad outputs become training data) | Owner-validation gate: only accept feedback-labeled trajectories. Verifier-second-pass before re-training. |

## Self-Bootstrapping Architecture (V13.x → V14)

The owner's dream of "ship V1 → V2 → ... → V10000 autonomously" requires a **closed loop**
where Surrogate-1's own outputs become Surrogate-1's training data. Diagram:

```
[idea]                                                  [V_n weights]
   |                                                          ^
   v                                                          |
[Surrogate-1 inference run] -> [trajectory log] -> [verifier] -|
        ^                              |               |
        |                              v               v
   [tools: shell/edit/test]   [ADP-format trace]  [outcome: pass/fail/owner-accept]
                                       |               |
                                       v               v
                                 [storage: HF private dataset 'surrogate-1-trajectories']
                                       |
                                       v
                              [next-version trainer reads this as merge_external line]
```

Components to build (V13.1 deliverables, not V13.0 scope):
1. Inference logger — patches OpenHands runtime to dump every action/obs to JSONL.
2. Verifier worker — runs unit tests + lint + owner-accept polling, labels each trajectory.
3. Curator pipeline — filters, dedupes, uploads to `axentx/surrogate-1-trajectories`.
4. Cron — weekly retrain trigger when new-trajectory count > 1000 since last run.

**Per Phase 5 self-training rule**: this is the natural next step. V13.0 = data + critic.
V13.1 = bootstrap loop. V14 = multimodal computer-use closure.
